## Auth Pattern (Critical)
- **Firebase Auth portals** (restaurant, rider, admin): Do NOT use `localStorage.removeItem()` on logout. Firebase Auth manages the session internally — no localStorage involvement.
- **Customer portal** (`sign-in.html`): Uses Firestore password auth. Session stored in `wawa_user`, cart in `wawa_cart` — this is correct.
- **Sign-in redirect pattern**: Poll `firebase.auth().currentUser` every 200ms, redirect only after confirmed non-null user. Safety net: 2.5s timeout.
