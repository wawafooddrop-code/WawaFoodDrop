## Today's Work — Rider ↔ Restaurant Handover Flow

### The Problem
The two-party handover between rider and restaurant needed a clear handshake so that neither party could skip the other. Multiple iterations were needed to get the flow right.

### Final Working Flow

#### Order Status Definitions
| Status | Who sees it | Meaning |
|--------|-------------|---------|
| `placed` | Restaurant: Incoming | Customer placed order |
| `accepted` | Restaurant: Preparing | Restaurant accepted |
| `preparing` | Restaurant: Preparing | Kitchen is cooking |
| `ready_for_pickup` | Restaurant: Ready tab | Food is ready |
| `picked_up` | Both — in Available (rider) / Ready (restaurant) | Rider has picked up, waiting for restaurant handover confirmation |
| `out_for_delivery` | Rider: Active Orders | Restaurant confirmed handover, rider is delivering |
| `delivered` | — | Order complete |
| `cancelled` | — | Cancelled (not in customer tracker yet) |

#### Rider Portal (`rider-available.html` — Available Orders tab)
Query watches: `['preparing', 'ready_for_pickup', 'picked_up']`

| Status | Condition | Badge | Button |
|--------|-----------|-------|--------|
| `preparing` | unclaimed | "Headed to Restaurant" | **Claim Order** |
| `preparing` | own (claimed) | "Claimed" | ~~Claimed~~ (grey) |
| `ready_for_pickup` | own OR unclaimed | "Ready for Pickup" | **Pick Up** |
| `picked_up` | own | "Waiting to Handover" | ~~Waiting to Handover~~ (grey) |

**Key:** On "Pick Up" click — must set BOTH `status: 'picked_up'` AND `riderId` + `riderName`. Previously `riderId` was NOT being set for unclaimed orders, causing the order to vanish after pickup.

#### Restaurant Portal (`restaurant-orders.html` — Ready tab)
Ready tab filter: `ready_for_pickup || picked_up`

| Status | Button |
|--------|--------|
| `ready_for_pickup` | "Awaiting Rider Pickup" (grey, disabled) |
| `picked_up` | **"Handover"** (green, clickable → `out_for_delivery`) |

#### Rider Portal (`rider-active.html` — Active Orders tab)
Query watches: `['out_for_delivery']` only

| Status | Button |
|--------|--------|
| `out_for_delivery` | **"Mark as Delivered"** |

**Key:** `picked_up` orders are NOT in Active Orders. They stay in Available Orders until restaurant taps "Handover".

### Scenario A — Restaurant acts FIRST (no rider claimed yet)
1. Restaurant marks `ready_for_pickup` → Ready tab shows "Awaiting Rider Pickup" (grey)
2. Rider sees unclaimed `ready_for_pickup` in Available → taps "Pick Up"
3. `picked_up` + `riderId` set → order turns **"Waiting to Handover"** (grey, stays in Available)
4. Restaurant "Handover" button activates → taps it → `out_for_delivery`
5. Order moves to rider's Active Orders → rider taps "Mark as Delivered" ✅

### Scenario B — Rider claims FIRST
1. Rider claims `preparing` → `riderId` set → restaurant sees rider name
2. Restaurant marks `ready_for_pickup` → order stays assigned to that rider
3. Rider sees own `ready_for_pickup` in Available → taps "Pick Up"
4. `picked_up` + `riderId` set → **"Waiting to Handover"** (grey, stays in Available)
5. Restaurant "Handover" activates → taps it → `out_for_delivery`
6. Order moves to rider's Active Orders → rider taps "Mark as Delivered" ✅

## Customer Portal — Order Tracker (`order-success.html`)

5-step tracker, polls every 5 seconds, maps internal statuses to customer-facing steps:

```js
const STATUS_TO_STEP = {
  'placed': 0, 'pending': 0, 'accepted': 0,   // → Order Confirmed
  'preparing': 1,                              // → Preparing
  'ready_for_pickup': 2,                        // → Ready for Pickup
  'picked_up': 3, 'out_for_delivery': 3,       // → Out for Delivery
  'delivered': 4                                // → Delivered
};

const STATUS_STEPS = [
  { key: 'confirmed',         label: 'Order Confirmed',   icon: 'check',            desc: '...' },
  { key: 'preparing',         label: 'Preparing',          icon: 'restaurant',       desc: '...' },
  { key: 'ready',             label: 'Ready for Pickup',   icon: 'local_shipping',   desc: '...' },
  { key: 'out-for-delivery',  label: 'Out for Delivery',   icon: 'delivery_dining',  desc: '...' },
  { key: 'delivered',         label: 'Delivered',           icon: 'home',             desc: '...' }
];
```
